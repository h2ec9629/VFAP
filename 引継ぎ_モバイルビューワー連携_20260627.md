# 引継ぎ：モバイルビューワー連携 (2026-06-27)

## 概要
VFAPの日程・配送情報をスマートフォンで閲覧できるモバイルビューワーを新規作成。
VFAPの計算済み描画データをGist経由でモバイルに流す設計。

---

## 完成した仕組み

```
VFAP（保存ボタン or 加工記録更新）
  │
  ├─ nittei.js: computeGanttRender() → {rows, gantt_render} を8502サーバーへPOST
  │
  ├─ app.py _SaveHandler.do_POST() → nittei.json（rowsのみ）+ gantt_render.json 保存
  │
  ├─ _gist_push_async() → nittei_to_gist.py をバックグラウンド実行
  │                         gantt_render.json を読んでGistへpush
  └─ （加工記録更新時）_update_gantt_render() → gantt_render.json をPython側で再計算

Gist (reminder_sync.json)
  ├─ reminders:      既存リマインダー（保持）
  ├─ excel_schedule: 引取・納品日程（MM/DD→YYYY-MM-DD変換済み）
  ├─ gantt_render:   VFAPの計算済みガントデータ（BAR_OFFSET適用済み）
  └─ synced_at:      同期日時

mobile_viewer.html → Gistから gantt_render を受け取りそのまま描画（再計算ゼロ）
```

## 設計上の重要ポイント

### BAR_OFFSET問題（このプロジェクトの発端）
- 旧方式: nittei_to_gist.py が nittei.json から位置を再計算 → BAR_OFFSET（業務開始時間）が不明でズレ発生
- 新方式: VFAPのJS側で BAR_OFFSET を適用した startH を計算済みで gantt_render.json に書き出し → モバイルは受け取るだけ

### 8502サーバーのリロード注意
- 8502 HTTP サーバーは app.py 起動時に一度だけ立ち上がるデーモンスレッド
- app.py を編集後は必ず **VFAP再起動（start.bat）** が必要（ページリロードでは反映されない）

### Gistキャッシュ問題
- `gist.githubusercontent.com/raw/...` URL は CDN キャッシュが強い
- mobile_viewer.html では `api.github.com/gists/{id}` を使用して解決済み
