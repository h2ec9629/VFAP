## ⚠️ 地雷メモ（厳守）

1. **index.lock / *.lock が頻発する**
   git操作のたび `fatal: Unable to create '...\.git\index.lock': File exists` が出た。
   → `del .git\index.lock`（または `del .git\*.lock`）してから再実行。
   しつこく復活する場合はVS Code/SourceTree/GitHub Desktop等がgitを握っている疑い → 閉じる。

2. **コミットメッセージは英語推奨**
   日本語メッセージをコマンドプロンプトに貼ると文字化け・半角スペース混入で誤解釈の事故あり。`-m "english msg"` が安全。vimは出さない。

3. **app.py は LF/CRLF ノイズで丸ごと "M" 表示される**
   `git diff` が全行差分に見えても中身は同じ（autocrlf）。慌てない。

4. **サンドボックスからは .git を書けない**（Operation not permitted）
   git の add/commit/merge/stash apply 等はすべて**おじさんの手元コマンド**で実行。
   ※ ファイル本体（app.py等）の編集・3wayマージのファイル生成はサンドボックスで可能。

5. **Edit/Writeツールが app.py / HTML の末尾を切る**
   app.py編集は bash+Python文字列置換 → `tail`末尾確認 → `python3 -m py_compile`。

6. **8503バックグラウンドサーバは古いコードが残る**
   app.py変更後はアプリ再起動前に `taskkill /F /IM python.exe` → 起動 → `curl http://localhost:8503/chohyo_ping` で `{"ok":true,"ver":"chohyo1"}` 確認。
