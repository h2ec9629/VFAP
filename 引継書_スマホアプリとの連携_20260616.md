## 現状のデータフロー（移行前）

```
SGT VBA → push-to-gist.ps1 → Gist (excel_schedule) → モバイルアプリ配送タブ
```

PS1（毎晩23:45）がGistの `excel_schedule` を保持しながら
ObsidianリマインダーデータをマージしてGistに書き戻す構造。

---

## 移行後のデータフロー（目標）

```
VFAPの日程ページで進捗更新
    ↓ 保存ボタン（saveRows）
app.py の8502ハンドラ
    ├─ nittei.json 書き込み（既存処理）
    └─ バックグラウンドスレッドでGistプッシュ（追加予定）
           ↓（1〜2秒後）
    Gist の excel_schedule を上書き
           ↓
    モバイルアプリの配送タブに反映
```

SGTが `excel_schedule` を書かなくなるため、VFAPが直接上書きしても競合しない。
モバイルアプリ側の変更は**不要**。

---

## データ変換仕様

`nittei.json` → `excel_schedule` への変換ルール：

| nittei.json | excel_schedule | 備考 |
|---|---|---|
| `ac` | `ac_side[].date` | 引取日。"06/11" → "2026-06-11" に変換 |
| `nm` | `ac_side[].ag` | 品名 |
| `ad` | `ad_side[].date` | 納品日。同上 |
| `nm` | `ad_side[].d` | 品名 |

日付変換: `"MM/DD"` → 年は実行時の年を付与（年をまたぐ場合は要注意）

---

## 実装方針

- **トリガー**: 日程ページの「保存」ボタン（`saveRows()`）が走ったタイミング
- **追加箇所**: `app.py` の8502ポートハンドラ内、nittei.json書き込み直後
- **非同期処理**: `threading.Thread` でバックグラウンド実行（保存レスポンスをブロックしない）
- **Gist更新方法**: 現在のGistを取得 → `excel_schedule` のみ差し替えて `PATCH`
  （`reminders` や `synced_at` は保持する）
